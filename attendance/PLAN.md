# 정기청소 용역자 근무·정산 관리 툴 — 빌드 계획

2026-07-17 확정. 젠스파크AI가 정리한 일반적인 "직원 관리 프로그램" 계획안을 검토했으나, 실제 사업 구조와 맞지 않는 부분(주52시간, 연차, 4대보험, GPS)이 있어 아래와 같이 재정의함.

## 1. 사업 구조 (전제)

- 작업자는 **비정규직 근로자가 아니라 외주 용역자** — 근로계약서가 아니라 외주 용역 계약서.
- **4대보험 미적용**, **사업소득 3.3% 원천징수**만 적용.
- 정산 흐름: **매니저가 작업자별 근무시간을 확정 → 대표(경리부)가 승인 → 익월 10~15일 지급.**
- **연차, 주52시간 근무제, 4대보험 등 근로기준법 관련 개념은 다루지 않음.**

## 2. 기술 스택 (확정)

기존 클린매니저와 동일 스택 유지 — 별도 인프라를 새로 만들지 않고 나중에 클린매니저에 합치기 쉽게.

- **Vite + React** (클린매니저 루트 앱과 동일)
- **Firebase** — `clean-manager-60bc9` 프로젝트 공유 (Firestore/Auth), 클린매니저의 기존 직원·팀 데이터 참조 가능
- 레포: 클린매니저와 같은 GitHub 레포, `attendance/` 폴더에 독립 프로젝트로 존재 (dev 포트 5176)
- 젠스파크 안(Next.js+Prisma+PostgreSQL+Vercel/Neon)은 **채택 안 함**

## 3. 범위 (MVP)

**포함:**
- 인증/조직 — 클린매니저 기존 로그인 체계와 연동 (매니저/대표 계정 구분)
- 작업자 마스터 — 외주 용역자 프로필(이름/연락처/계약 정보), 클린매니저 직원 데이터와 연계 고려
- 근무기록 — **GPS/Geo-Fencing 없이** 매니저가 직접 입력/확정하는 방식 (현장/일자별 근무시간)
- 정산 승인 흐름 — 매니저 확정 → 대표 승인 → 지급 상태 관리 (익월 10~15일 지급 기준)
- 3.3% 원천징수 계산이 포함된 정산 내역
- 관리자 대시보드 — 정산 현황, 승인 대기 목록 등

**제외 (젠스파크 안에는 있었지만 우리 사업 구조엔 불필요):**
- 주52시간 자동계산 (`week52.ts` 류 로직)
- 연차·휴가 관리
- 4대보험 처리
- GPS 기반 출퇴근 체크(Geo-Fencing), PWA 위치 인증
- 교대 시프트, 성과평가, AI 인사이트 (Phase 2 성격 — 지금 논외)

## 4. 프로젝트 구조

지금은 규모가 작은 placeholder 단계라 `CLAUDE.md` 라우팅 파일 + `.claude/rules/` 분산 구조는 **적용 안 함** — 규칙이 늘어나면 그때 도입.
